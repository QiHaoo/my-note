容器和虚拟机面对相同的问题，使用的也都是虚拟化技术，只是所在层次不同。

## 共同点

- 目的都是隔离资源、保证系统安全
- 都尽量提高资源的利用率
- 内外两个系统都完全隔离、互不干扰

## 实现层次的差异

虚拟机（VirtualBox / VMware / Hypervisor）能在宿主机里完整虚拟化出一套计算机硬件，在里面还可安装任意操作系统。数据中心的 Hypervisor 可以把一台物理服务器虚拟成多台逻辑服务器，彼此独立、按需分隔资源。

两者的分层结构对比：

- **虚拟机**：应用 → Guest OS → Hypervisor → 物理硬件
- **容器**：应用 → Docker / 容器 → 宿主机 OS / 内核 → 物理硬件

从实现角度看：

- **虚拟机**虚拟化出来的是硬件，需要在上面再装一个操作系统才能运行应用。硬件虚拟化和操作系统都比较"重"，消耗大量 CPU、内存、硬盘，属于"重复劳动"。好处是隔离程度非常高，虚拟机之间完全无干扰。
- **容器**直接利用下层硬件和操作系统，比虚拟机少一层，节约 CPU 和内存，非常轻量，能更高效利用硬件。代价是多个容器共用内核，隔离程度没有虚拟机高。

> Docker 官网图示其实并不太准确：容器并不直接运行在 Docker 上，Docker 只是辅助建立隔离环境，让容器基于 Linux 操作系统运行。

## 运行效率对比

运行效率是容器相比虚拟机最大的优势。同样的系统资源，虚拟机只能跑 3 个应用，其他资源都用来支持虚拟机运行；而容器能把这部分资源释放出来，同时运行 6 个应用（此为形象展示，非严谨数值比较）。

| 维度 | 虚拟机 | 容器 |
|------|--------|------|
| 体积 | 普通 Ubuntu 虚拟机 GB 级别，装些应用易上到 10GB | Ubuntu 镜像只有几十 MB |
| 启动时间 | 通常需要几分钟 | 基本上不超过一秒钟 |
| 并发量 | 同时运行十来个虚拟机可能就是极限 | 同时跑上百个容器毫无问题 |

## 两者结合使用

虚拟机和容器不是互相排斥的，完全可以结合使用：用虚拟机实现与宿主机的强隔离，再在虚拟机里用 Docker 容器快速运行应用（正如本课程的做法）。

## 关于"容器就是轻量级的虚拟机"

- 从比喻角度看，容器不是虚拟机，但在隔离方面又与虚拟机很像，因此"容器就是轻量级的虚拟机"这种说法可以接受。
- 更严谨地说：容器本质是一个带着一组"隔离与限制"的进程，与虚拟机差得很远；虚拟机虚拟出一套软硬件系统环境，而容器只是一个进程。两者相同点只在隔离性这个广义角度上类似。
