## 容器到底是什么

容器（Container）好比现实世界的集装箱：标准化封装货物，隔离箱内箱外两个世界。计算机世界里容器封装的"货物"是运行中的应用程序（进程），同样把进程与外界隔离开。

通过实际操作可以观察容器内部：

- `docker pull alpine` 拉取镜像，`docker run -it alpine sh` 进入容器
- 容器内 `cat /etc/os-release` 看到的是 Alpine Linux 3.15，而非宿主机的 Ubuntu；`ps` 只能看到一个"干净"的环境
- 进入 `ubuntu:18.04` 容器后可执行 `apt update`、`apt install` 等任意操作，但都不会影响外面的系统

> 容器就是一个特殊的隔离环境，它能让进程只看到这个环境里的有限信息，不能对外界环境施加影响。

## 为什么要隔离

隔离的出发点是系统安全，同时配合资源配额限制进程对硬件的占用：

| 目的 | 说明 |
|------|------|
| 系统安全 | 不受限的进程能看到所有文件、进程、网络、内存，恶意或有 Bug 的程序易搞瘫系统或泄漏信息；Linux 用户权限机制较薄弱，与真正"隔离"相差很远 |
| 资源隔离 | CPU、内存、硬盘、网卡等资源有限，为应用切分配额（如只用一个 CPU、1GB 内存），避免某个应用无限制占用 |

容器让应用运行在严密防护的"沙盒"（Sandbox）内，进程可自由活动但绝不允许"越界"。

## 与虚拟机的区别

容器和虚拟机面对相同的问题（隔离资源、保证安全、提高利用率），都使用虚拟化技术，只是所在层次不同：

| 对比维度 | 虚拟机 | 容器 |
|----------|--------|------|
| 虚拟化目标 | 硬件（Hypervisor 虚拟出多台逻辑服务器） | 直接利用下层硬件和操作系统 |
| 运行层次 | 虚拟硬件后需再装一个操作系统 | 比虚拟机少一层，更轻量 |
| 隔离程度 | 非常高，虚拟机间完全无干扰 | 共用内核，隔离程度不如虚拟机 |
| 资源开销 | 硬件虚拟化 + OS 都"重"，消耗大 | 消耗很少，运行效率是最大优势 |
| 体积 | GB 级，易上到 10GB | 镜像只有几十 MB |
| 启动时间 | 几分钟 | 不超过一秒 |

虚拟机和容器并不互斥：可用虚拟机实现与宿主机的强隔离，再在虚拟机里用 Docker 容器快速运行应用。

## 隔离是怎么实现的

容器的奥秘在于 Linux 内核为资源隔离提供的三种技术，它们初衷并非为容器而生，结合起来却发生了奇妙的"化学反应"：

| 技术 | 出现时间 | 作用 | 比喻 |
|------|----------|------|------|
| namespace | 2002 年 Linux 2.4.19 | 隔离文件系统、主机名、进程号、网络等资源空间 | 给进程盖了一间小板房 |
| cgroup | 2008 年 Linux 2.6.24 | 限制进程的 CPU、内存等资源配额和优先级 | 给小板房加了天花板 |
| chroot | 1979 年 UNIX V7 | 更改进程根目录，限制访问文件系统 | 给小板房铺上了地砖 |

一句话概括容器：**普通进程 + namespace + cgroup + chroot = 特殊的进程 = 容器**。
