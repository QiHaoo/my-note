## 我的 Kubernetes 学习经验

Kubernetes 是一个全新的系统，入门时面对一大堆不知所云的概念（Pod、ConfigMap、Deployment 等），几乎没有以往经验可借鉴。下面是一些实践中总结的建议：

| 建议 | 说明 |
|------|------|
| 定个小目标 | 结合实际定明确目标（如"搞明白 Nginx Ingress Controller 的用法"），有方向有重点，便于检查进度、集中精力，否则易迷失在知识海洋 |
| 翻越概念高山 | 面对陌生概念必须反复阅读思考、做实验加深印象，度过"危险期"打好基础后学习会轻松许多 |
| 一定要做实验 | Kubernetes 是"操作系统"，只看文字资料"纸上谈兵"难以弄清工作原理和运行机制，必须上手演练 |
| 选对实验工具 | 入门用 minikube（简单方便、功能齐全）；深入后应改用 kubeadm，才能更透彻研究（minikube 基于 Docker 虚拟，在节点和网络等方面离真实环境有差距） |
| 勤记笔记 | "好记性不如烂笔头"，及时把思考和体会写下来，不必强求完整，一两句话、几个链接也有价值 |
| 画思维导图 | 笔记积累到一定数量后归纳浓缩，用树状发散形式整理思路，把碎片化笔记与有结构的思维导图结合 |

## 今后要怎么继续学

课程结束只是"万里长征第一步"，可以从以下四个方向继续深入：

| 方向 | 说明 |
|------|------|
| 阅读官网文档 | 资料丰富详细，覆盖入门、安装、概念、教程、运维、参考手册。优点是全面权威；缺点是太多太杂、不完全面向初学者，要善用搜索用关键字快速定位 |
| 看官方博客 | 文档只描述现状，博客能看到 API 对象怎么设计、怎么发展到今天。推荐从 2015 年第一篇起简略过一遍，"以史为鉴"理解技术决策与演变过程 |
| 逛 CNCF 全景图 | 找感兴趣的项目在 K8s 环境里部署、在实践中学习。CNCF 项目分毕业（Graduated，最成熟、可用于生产）、孵化（Incubating）、沙盒（Sandbox，实验性质）三类 |
| 参加培训认证 | 官方认证国内较熟知的是 CKA（Certified Kubernetes Administrator），更高级的是 CKS。CKA 重动手操作、用 kubectl 解决实际问题，考点覆盖面广、考试时长 2 小时，且有三年时效期，过期需重考 |

> 终点，更是起点。打好基础后，Kubernetes 的世界才刚刚向你展开。
